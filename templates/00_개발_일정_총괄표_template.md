## 6. 의존성 체인 (Critical Path)

```
Day 1-2: ERD 확정 + AGENTS.md 통일
   └─> Day 3-4: Entity·Repository (Lead 선행)
         └─> Day 5: 각 모듈 Service 시작 (병렬 4명)
               └─> W2: Controller + View
                     └─> W3 초: 통합 + 외부 API
                           └─> W3 중: 부하 테스트 + 디버깅
                                 └─> W3 말: 운영 배포 + 시연 리허설
```

**Critical Path 단축 전략 (3주 압축용)**

- **W1 화요일까지** ERD 확정. 늦으면 전체 일정 깨짐.
- AI 에이전트로 Entity / Repository / 단위 테스트 골격 W1 후반 자동 생성 → 모듈별 Service 시작 1일 앞당김.
- **외부 API spike**: Lead가 W1 후반에 외부 API 1일 spike → W2 본 연동에서 막힘 최소화.
- **DevOps/QA가 W1에 CI/CD 완성** → 모든 PR이 빠른 게이트 가짐.

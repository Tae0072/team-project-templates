## 5. 의존성 체인 (Critical Path)

```
ERD 확정
   └─> Entity 생성 (Lead)
         └─> Repository 테스트
               └─> Service 구현 (각 담당자)
                     └─> Controller + Mustache
                           └─> E2E 시나리오
                                 └─> 배포
```

**Critical Path 단축 전략:**
- W1 후반 ERD 확정 → W2 월요일 Entity/Repo 오픈
- 외부 API는 W3 중반 **책임개발자 선행 착수**, W4 UI 통합
